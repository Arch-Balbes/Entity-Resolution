# ML Pipeline: Entity Resolution

Пайплайн для задачи **Entity Resolution** в маркетплейсе: определить, какие `profile_id` относятся к одному реальному человеку.

Состоит из двух частей:

1. **Обучение** — CatBoost на pairwise-признаках
2. **Streamlit-приложение** — загрузка новых данных и автоматический поиск групп дублей

## Streamlit-приложение: функционал

Веб-интерфейс для **практического поиска дублей** в произвольном файле с событиями. Приложение не показывает метрики качества модели на train/test — оно принимает ваши данные и возвращает готовые группы профилей.

### Назначение

В исходных данных одна строка — **событие** профиля, а не уникальный пользователь. Один человек может иметь несколько `profile_id`. Приложение:

- читает файл с событиями;
- находит пары похожих профилей;
- оценивает вероятность дубля моделью CatBoost;
- объединяет уверенные пары в **кластеры** (группы одного человека).

### Пошаговый сценарий

| Шаг | Действие |
|-----|----------|
| 1 | Запустить приложение (`docker compose up app` или `streamlit run app/streamlit_app.py`) |
| 2 | Открыть http://localhost:8501 |
| 3 | В боковой панели выбрать **режим объединения** (см. ниже) |
| 4 | Загрузить **CSV** или **Parquet** с событиями |
| 5 | Нажать **Найти профили одного человека** |
| 6 | Просмотреть таблицу групп и скачать результаты |

Если модель ещё не обучена, приложение покажет ошибку с командой для запуска `docker compose run --rm train`.

### Входные данные

Формат совпадает с обучающим датасетом (`split_label_train_V3`).

**Обязательные колонки** (одна строка = одно событие):

| Колонка | Описание |
|---------|----------|
| `profile_id` | Идентификатор профиля |
| `created_at` | Время события |
| `first_name`, `last_name`, `email`, `phone`, `birthday`, `sex` | Персональные поля |
| `non_processing_features` | Список технических признаков события |
| `realtime_features` | JSON с гео и контекстом |
| `fs_features` | Список признаков feature store |

**Необязательные:** `entity_id`, `entity_type` — нужны только для обучения и оценки, при инференсе не требуются.

Поддерживаются форматы **`.csv`** и **`.parquet`**. Для CSV списковые поля парсятся автоматически.

### Внутренний пайплайн обработки

```text
Загруженный файл
    → нормализация пропусков (NA, unknown в поле sex)
    → агрегация событий в таблицу профилей (1 строка = profile_id)
    → blocking: кандидатные пары (~до 250 000)
         • совпадение телефона
         • email_domain + имя / префикс телефона
         • гео + имя / пол
         • редкие пересечения fs_features
    → CatBoost: score (вероятность дубля) для каждой пары
    → классификация пары: auto_merge / manual_review / reject
    → граф: связные компоненты по выбранным парам
    → кластеры с cluster_id (person_00001, …)
```

### Режимы объединения (боковая панель)

| Режим | Поведение | Когда использовать |
|-------|-----------|-------------------|
| **auto_merge + manual_review** | Склеиваются пары с высоким score и пары для ручной проверки | По умолчанию; даёт больше найденных групп |
| **только auto_merge** | Только пары с очень высоким score, без жёстких конфликтов и со сильным evidence | Минимум ложных объединений |
| **по порогу score** | Слайдер: все пары с score ≥ порога попадают в граф | Тонкая настройка под ваши данные |

Для пары в зону **auto_merge** дополнительно проверяется: нет конфликта по дате рождения / полу / префиксу телефона и есть сильное evidence (совпадение телефона, email или высокий Jaccard по техническим признакам).

### Результаты на экране

После обработки отображаются метрики запуска:

- число **событий** и **профилей**;
- число **групп** (кластеров из 2+ профилей);
- число **профилей**, попавших в группы.

**Таблица кластеров** содержит, в частности:

| Поле | Смысл |
|------|-------|
| `cluster_id` | Идентификатор группы (`person_00001`, …) |
| `profile_id` | Профиль в этой группе |
| `cluster_size` | Размер группы |
| `first_name`, `last_name`, `email`, `phone`, … | Атрибуты профиля для проверки |

### Экспорт

| Файл | Содержимое |
|------|------------|
| `entity_clusters.csv` | Все профили с назначенным `cluster_id` — основной результат |
| `scored_pairs.csv` | Все кандидатные пары: `left_profile_id`, `right_profile_id`, `score`, `action` |

### Ограничения

- Обработка больших файлов (десятки тысяч профилей) может занять **1–3 минуты**.
- Blocking не перебирает все пары профилей (квадратичная сложность), поэтому часть истинных дублей может не попасть в кандидаты.
- Максимальный размер автоматического кластера — **20** профилей (защита от «снежного кома» ложных связей).
- Приложение **не перезаписывает** `entity_id` в исходных данных — только выдаёт предложенные группы для ручной или последующей автоматической обработки.

## Обучение модели

### Подготовка данных

Положите датасет в `data/`

Приоритет при авто-поиске: **parquet** → **csv**.

### Docker

```bash
docker compose build
docker compose run --rm train    # обучение (один раз)
docker compose up app            # Streamlit на http://localhost:8501
```

После обучения в `artifacts/`:

| Файл | Назначение |
|------|------------|
| `model.cbm` | CatBoost для инференса |
| `feature_columns.json` | Порядок признаков |
| `thresholds.json` | Пороги auto_merge / manual_review |
| `metrics.json` | Метрики на test (для отладки, не показываются в UI) |

### Локальный запуск

```bash
pip install -r requirements.txt
python scripts/train.py
streamlit run app/streamlit_app.py
```
